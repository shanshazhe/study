# 深度学习与人工智能学习路线（详细版）

> 适用人群：数学基础扎实、编程为全栈、英语交流无障碍  
> 目标：能独立完成从数据处理 → 模型训练 → 部署上线的完整闭环

---

## 总览（建议 16-24 周）

- **学习节奏**：每周 8-15 小时（可按时间缩放）
- **交付物**：每阶段 1 个可运行项目 + 1 份总结笔记
- **核心栈**：Python / PyTorch / FastAPI / Docker / Git

---

## 0. 环境与工具（1 周）

- **目标**：搭建可靠的实验与工程环境
- **内容**：
  - Python 环境管理（venv/conda）
  - PyTorch、Jupyter、基础依赖包
  - GPU 可用性确认（如有）
  - 项目模板：日志、配置、随机种子、可复现实验
  - Git 工作流与项目结构

**交付物**：一个“训练脚手架”项目（含训练/评估/日志）

---

## 1. 机器学习基础（2-3 周）

- **核心概念**
  - 训练/验证/测试划分与数据泄露
  - 过拟合/欠拟合、偏差-方差
  - 评价指标（分类/回归）

- **模型与方法**
  - 线性/逻辑回归
  - 决策树、随机森林、GBDT
  - KNN、K-Means、PCA

- **实践**
  - sklearn 实验与调参
  - 特征工程与数据清洗

**交付物**：中小数据集建模报告（含指标对比与结论）

---

## 2. 深度学习核心（4-6 周）

- **基础能力**
  - 张量与 Autograd
  - 前向/反向传播
  - 常见损失函数与优化器
  - 初始化、正则化、归一化

- **经典结构**
  - MLP（全连接）
  - CNN 基础与卷积直觉
  - RNN/LSTM/GRU（理解序列建模基础）

- **训练技巧**
  - 学习率调度、早停、梯度裁剪
  - 过拟合排查与可视化诊断

**交付物**：
- 手写数字识别（CNN）
- 文本/时间序列入门任务（RNN 或 LSTM）

---

## 3. Transformer 与现代架构（3-4 周）

- **核心概念**
  - 注意力机制、自注意力
  - 位置编码、残差与层归一化
  - 编码器/解码器结构

- **实践**
  - 使用 Transformer 做文本分类或翻译简化版
  - 训练一个小型语言模型（小数据/小参数）

**交付物**：一个基于 Transformer 的小项目

---

## 4. 方向专项（任选 1-2 个，4-8 周）

### 方向 A：计算机视觉（CV）
- 目标检测与分割基础
- 常见模型：ResNet / U-Net / YOLO（理解与复现）
- 项目示例：道路裂缝检测、商品识别

### 方向 B：自然语言处理（NLP）
- 词向量与 Tokenizer
- BERT / GPT 思想与微调流程
- 项目示例：情感分析、文本分类、问答系统雏形

### 方向 C：时间序列
- 序列特征、趋势与季节性
- 基线模型 + LSTM/Transformer
- 项目示例：销量预测、异常检测

---

## 5. 工程化与部署（3-4 周）

- **模型工程**
  - 配置化训练、日志、实验管理
  - 数据版本与模型版本
  - 训练/推理分离

- **部署**
  - FastAPI 模型服务
  - Docker 镜像打包
  - 简单灰度与监控

**交付物**：可部署的模型服务（本地或云端）

---

## 6. 综合项目（2-4 周）

选择 1 个方向，完成完整闭环：

- 数据采集/清洗
- 模型训练与评估
- 推理服务与简单前端展示
- 项目总结（技术方案 + 结果 + 改进点）

**项目示例**：
- 图片内容审核（CV）
- 智能客服 FAQ 机器人（NLP）
- 设备故障预测（时间序列）

---

## 7. 12 周加速版（按周拆解）

**第 1 周**：环境 + 训练脚手架  
**第 2 周**：sklearn 基础 + 指标体系  
**第 3 周**：特征工程 + 模型对比  
**第 4 周**：PyTorch 基础 + MLP  
**第 5 周**：CNN + 图像任务  
**第 6 周**：RNN/LSTM + 序列任务  
**第 7 周**：Attention + Transformer  
**第 8 周**：Transformer 小项目  
**第 9-10 周**：方向专项项目  
**第 11 周**：FastAPI + Docker 部署  
**第 12 周**：综合项目打磨与复盘

---

## 8. 输出要求（建议每阶段都做）

- 代码仓库（可复现）
- 技术笔记（概念 + 实验结论）
- Demo 或报告（能展示成果）
